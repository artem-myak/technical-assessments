# Аналітичний кейс: Аудит А/Б тесту та продуктове мислення

Цей звіт містить стратегічний та статистичний аналіз показників e-commerce платформи, зосереджений на оцінці експериментів та пошуку причин падіння конверсії.

---

## Частина 2: Аналіз А/Б тесту

### 2.1. Оцінка успішності тесту
**Висновок:** Ні, на основі наданих даних неможливо зробити однозначний висновок про успішність тесту.

**Обґрунтування:**
* **Відсутність статистичної значущості (p-value):** Хоча ми спостерігаємо приріст конверсії на 0.25%, без розрахунку p-value ми не знаємо, чи є це результатом змін, чи просто випадковим шумом.
* **Невідома тривалість тесту:** Для врахування тижневої сезонності тест має тривати мінімум 7–14 днів. Без цієї інформації дані можуть бути спотворені випадковими факторами конкретного дня.

### 2.2. Дані, яких не вистачає для аналізу
Для прийняття обґрунтованого бізнес-рішення потрібні:
* **Статистична значущість та довірчі інтервали:** Щоб підтвердити, що приріст не є випадковим.
* **MDE (Minimum Detectable Effect):** Щоб розуміти, чи достатньо поточної вибірки (по 10 тис. у групі) для фіксації різниці у 0.25%.
* **Розподіл за сегментами:** Порівняння результатів для різних платформ (iOS, Android, Web) та типів користувачів.

### 2.3. Додаткові метрики для відстеження
Окрім основної конверсії, варто контролювати:
* **AOV (Середній чек):** Щоб переконатися, що приріст замовлень не призвів до падіння їхньої вартості.
* **Revenue per User (Дохід на користувача):** Фінальна метрика ефективності для бізнесу.
* **Час до покупки:** Чи стала нова кнопка швидшим шляхом до замовлення.
* **CTR самої кнопки:** Для оцінки ефективності саме елемента інтерфейсу.

## Частина 3: Практичне мислення (Аналіз падіння конверсії)

### 3.1. Фреймворк дослідження падіння на 15%

Моє дослідження базується на принципі «від простого до складного» з використанням сегментації.

#### Крок 1: Технічний чекап та сегментація
* **Платформи:** Порівняння CR на iOS, Android та Web. Падіння лише на одній платформі вказує на технічний баг.
* **Джерела трафіку:** Перевірка, чи не було притоку нецільового трафіку, який «розмив» конверсію.
* **Логи помилок:** Перевірка стабільності платіжних шлюзів на етапі Checkout.

#### Крок 2: Аналіз воронки (Funnel Friction)
Пошук «вузького місця» на кожному мікро-кроці:
* **Головна → Кошик:** Чи працює кнопка додавання товару?
* **Кошик → Оплата:** Чи немає затримок при завантаженні форми реєстрації?
* **Оплата → Успішна покупка:** Чи коректно проходять транзакції?

#### Крок 3: Зовнішні та продуктові фактори
* **Маркетинг:** Чи не завершилася велика акція минулого тижня?
* **Ціноутворення:** Чи не було підвищення цін або вартості доставки?
* **Конкуренти:** Чи не запустили конкуренти агресивний розпродаж у цей період?
